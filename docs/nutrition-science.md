# КБЖУ и клетчатка: научная база расчёта

Справочник для нутрициологического движка Driftora. Собран из первичных источников:
**FAO** (FNP 77, INFOODS, Bognár), **USDA** (FoodData Central, Handbook 74, yield/retention tables),
**EFSA**, **WHO**, **IOM/NASEM (DRI 2005)**, **AOAC**, **Codex Alimentarius**,
**ТР ТС 022/2011** (маркировка ЕАЭС), **ГОСТ**, **Скурихин/Тутельян**, **МР 2.3.1.0253-21**,
рецензируемые журналы (AJCN, J Nutr, Nutrients, Br J Nutr, CVPR/ICCV). Полная библиография — в конце.

> Каждая цифра прослеживается до первоисточника. Там, где источник помечен как «требует сверки»,
> цифру перед хардкодом в код нужно подтвердить по оригиналу — эти места отмечены ⚠️.

## 0. TL;DR — главные решения

1. **Считать ккал самим из граммов по ОДНОЙ формуле, а не доверять полю `kcal` источника.** Курированные
   таблицы / USDA / OFF / FatSecret считают энергию по-разному (и по-разному трактуют клетчатку), поэтому
   одинаковый состав даёт разные ккал в зависимости от того, какое звено цепочки ответило. Единая формула —
   единственный способ держать КБЖУ внутренне согласованным. → §1

2. **Формула — коэффициенты ТР ТС 022/2011** (это ровно то, по чему обязана считаться любая российская
   этикетка): `ккал = 4·белок + 9·жир + 4·(углеводы−клетчатка) + 2·клетчатка [+ 2,4·сахароспирты + 7·этанол + 3·орг.кислоты]`.
   Для эритрита коэффициент 0. → §1

3. **Клетчатка = 2 ккал/г, не 0 и не 4.** «0» — устаревшая конвенция до 1990-х; современный консенсус
   (FAO / EU / FDA-2016 / ТР ТС) — 2 ккал/г. Главная ловушка: «углеводы» из USDA/OFF УЖЕ включают клетчатку —
   при показе российской строкой «углеводы отдельно, клетчатка отдельно» её надо вычесть, иначе двойной счёт. → §5

4. **Крупнейший источник ошибки — оценка массы порции, а не состав.** Ошибка массы переходит в ошибку ккал
   почти 1:1. Путаница «сырой ↔ готовый» даёт до 2–3× (рис ×2,9, паста ×2,6, гречка ×3,6). У нас это уже частично
   закрыто [`dryBasis.ts`](../server/src/nutrition/dryBasis.ts) — расширить yield-факторами. → §2, §6, §8

5. **Коэффициент пересчёта азота в белок (Jones) важен для орехов/семян/сои/какао/грибов** — общий 6,25 завышает
   белок (а значит и ккал) на 10–43% для этих категорий. Хранить `NFactor` на продукт. → §3

6. **Цель по клетчатке масштабировать от калорийного бюджета**, а не флэт-число: `max(25 г, бюджет/1000·10..14 г)`.
   РФ-норма 20–25 г (или 10 г/1000 ккал) — пол; IOM 14 г/1000 ккал — потолок. → §5, §9

7. **«≈ проверьте» — научно честная рамка, а не отговорка.** Даже фронтир-модели дают ~24–38% MAPE по ккал с
   одного фото — на уровне диетолога. Текст с весом («200 г творога 5%») снижает ошибку вдвое. Ширину «≈»
   привязать к качеству входа. → §10

8. **Атвотер завышает энергию цельных орехов (5–32%) и бобовых (8–16%)** — но эффект зависит от ФОРМЫ
   (цельный vs молотый), а не от вида. Опционально, keyed по форме. TEF/net-energy per-food НЕ моделировать —
   адаптивный TDEE по тренду веса уже это впитывает. → §7

## 1. Энергия из макросов — единая формула

### Каноническая формула для Driftora

```
ккал(100 г) = 4·белок_г
            + 9·жир_г
            + 4·углеводы_усвояемые_г        // = всего_углеводы − клетчатка − сахароспирты
            + 2·клетчатка_г
            + 2,4·сахароспирты_г            // эритрит = 0
            + 7·этанол_г                    // если учитывается
            + 3·орг.кислоты_г               // редко; только если поле есть отдельно
```

`углеводы_усвояемые` = углеводы БЕЗ клетчатки. Если источник даёт «всего углеводы» (USDA-стиль, включает
клетчатку) — вычесть клетчатку перед умножением на 4. Если источник уже даёт «усвояемые/net» — брать как есть.

### Почему именно эти коэффициенты: ТР ТС 022/2011, Приложение 4

Это не «один из вариантов Атвотера», а **закон**, по которому считается любая российская/ЕАЭС-этикетка.
Прочитано напрямую из официального PDF ЕЭК:

| Вещество | ккал/г | кДж/г |
|---|---|---|
| Белки | 4 | 17 |
| Углеводы (моно/ди-сахариды, кроме сахароспиртов) | 4 | 17 |
| Сахароспирты (кроме эритрита) | 2,4 | 10 |
| Эритрит | 0 | 0 |
| Жиры, жирные кислоты | 9 | 37 |
| Органические кислоты | 3 | 13 |
| Салатрим | 6 | 25 |
| Этанол | 7 | 29 |
| **Пищевые волокна** | **2** | **8** |

Этот набор совпадает с рекомендацией **FAO FNP 77** («extensive general factor system») и с
**Regulation (EU) 1169/2011, Annex XIV** почти дословно — то есть выбирая коэффициенты ТР ТС, мы одновременно
согласуемся и с международным консенсусом, и с тем, что напечатано на упаковке у российского пользователя.

> **Практический эффект:** если пользователь удивляется, что наше число не совпадает с этикеткой — это можно
> честно объяснить (в духе уже принятого стиля «Как это работает»): этикетка обязана считаться по фиксированным
> законодательным коэффициентам, которые отличаются и от «уточнённых» факторов Атвотера, и тем более от прямого
> лабораторного измерения. Три легитимных, но не идентичных подхода к одному продукту.

### Три способа посчитать «углеводы» и ловушка двойного счёта клетчатки

| Величина | Включает клетчатку? | Как считается |
|---|---|---|
| **Всего углеводы «по разности»** (USDA `Carbohydrate, by difference`, поле по умолчанию) | **Да** | `100 − вода − белок − жир − зола − этанол` |
| **Усвояемые углеводы «по разности»** | Нет | то же минус клетчатка |
| **Усвояемые углеводы прямым суммированием** (сахара + крахмал; предпочитает FAO) | Нет | HPLC/энзиматика |

- USDA `by difference` — это **остаток**, в который стекаются все ошибки измерения воды/белка/жира/золы плюс
  вся масса клетчатки плюс неучтённая органика (кислоты, танины). Это единственный макрос, чья точность ХУЖЕ,
  чем у входных величин. Отдельно измеренные сахара+крахмал+клетчатка часто НЕ сходятся с числом «по разности» —
  это норма, не баг данных.
- **Российско-европейская модель** (видно прямо в ТР ТС, Приложение 2) держит «усвояемые углеводы» и
  «пищевые волокна» ДВУМЯ параллельными, не вложенными строками, у клетчатки свой коэффициент 2 ккал/г.
- ⚠️ **Ловушка:** взяли «углеводы» из USDA как есть (в них уже сидит клетчатка) и рядом показали клетчатку
  отдельной строкой российского стиля → клетчатка посчитана дважды (внутри «углеводов» по 4 и отдельно). Либо
  вычитать клетчатку из USDA-числа перед показом, либо явно маркировать поле как «всего, включая клетчатку».

### Монозахаридные эквиваленты (на будущее)

Углеводы, выраженные как моносахаридные эквиваленты, используют **3,75 ккал/г (16 кДж/г)**, а «по весу» — 4,0.
Разница ~6,25%. Все потребительские базы (и вся наша цепочка) работают «по весу» → держимся 4,0 ккал/г и НЕ
смешиваем с 3,75.

### Куда ложится в коде

- [`server/src/nutrition/plausibility.ts`](../server/src/nutrition/plausibility.ts) — естественное место для
  recompute-проверки: посчитать `ккал_из_макросов` по формуле выше и сравнить с заявленной ккал источника.
  Расхождение > ~10–15% = флаг низкой уверенности / повод для «≈» (обычно это разный базис углеводов/жира,
  неучтённый спирт, либо реально необычный продукт).
- Обязательно применять вычет клетчатки: наивное `4·белок + 9·жир + 4·всего_углеводы` завышает энергию любого
  волокнистого продукта (овощи, бобовые, цельные злаки — ровно то, что логирует ЗОЖ-пользователь).

## 2. Иерархия ошибок — что реально двигает точность

Отсортировано по величине вклада. Это карта приоритетов: чинить сверху вниз.

| # | Источник ошибки | Типичная величина | Статус у нас |
|---|---|---|---|
| 1 | **Масса порции** (фото/текст без веса) | 1:1 в ккал; 50–65% оценок в пределах ±25–30% — это ПОТОЛОК даже у диетологов | «≈»-рамка есть; см. §8, §10 |
| 2 | **Сырой ↔ готовый (yield)** | до **2–3×** (рис ×2,9, паста ×2,6) | частично закрыто [`dryBasis.ts`](../server/src/nutrition/dryBasis.ts); расширить §6 |
| 3 | **Коэффициент N→белок** (орехи/соя/какао/грибы) | +10…+43% к белку и его вкладу в ккал | не реализовано; §3 |
| 4 | **Атвотер общий vs специфический** | до +38% для отдельных продуктов (клетчатка, орехи) | §1, §7 |
| 5 | **Метаболизируемая энергия цельных орехов/бобовых** | −5…−32% реально усваивается | опционально; §7 |
| 6 | **Коэффициент клетчатки 0 vs 2 vs 4 ккал/г** | относительно велика, абсолютно мала (клетчатка — меньшинство массы) | §5 |
| 7 | **Жир: total lipid vs NLEA/триглицеридный** | 0–4% (мясо/молочка/масло/орехи) … 15–40% (субпродукты, тощая рыба, злаки) | §4 |

**Вывод:** проблема активности/порции доминирует над всеми ошибками состава вместе взятыми. «Ошибка активности >
ошибок BMR» из памяти проекта — тот же принцип на стороне еды: точность оценки массы важнее точности любого
коэффициента в таблице.

## 3. Белок — коэффициент пересчёта азота (Jones / NFactor)

Ни Кьельдаль, ни Дюма не мерят белок напрямую — они мерят **общий азот**, который умножается на коэффициент.
Общий 6,25 предполагает 16% азота в белке; реально 13–19% → факторы Джонса (USDA Circular 183, 1941; в
терминологии INFOODS — `XN`). В коде USDA это поле `NFactor`; по умолчанию 6,25.

### Таблица (только то, что реально сдвигает результат)

| Категория | XN | Отклонение от 6,25 |
|---|---|---|
| Мясо, рыба, яйца, кукуруза, **большинство бобовых/фасоли** | 6,25 | 0% |
| Молоко/молочка | 6,38 | 6,25 **занижает** на 2,0% |
| Казеин/сыр | 6,40 | занижает на 2,3% |
| Рис | 5,95 | завышает на 5,0% |
| Пшеница цельная (рожь/ячмень/овёс/просо) | 5,83 | завышает на 7,2% |
| Пшеничная мука рафинированная | 5,70 | завышает на 9,6% |
| **Соя / соевые продукты** | 5,71 | завышает на 9,5% |
| Желатин | 5,55 | завышает на 12,6% |
| Дрожжи | 5,70 | завышает на 9,6% |
| **Орехи (общее) и все семена** | 5,30 | завышает на 17,9% |
| Миндаль | 5,18 | завышает на 20,7% |
| Кофе | 5,30 | завышает на 17,9% |
| **Какао/шоколад** | 4,74 | завышает на 31,9% |
| **Грибы** | 4,38 | завышает на 42,7% |

> **Важная поправка к «народному» знанию:** 5,30 — это про ОРЕХИ И СЕМЕНА, НЕ про бобовые вообще. Нут, чечевица,
> обычная фасоль, маш → 6,25. Только соя (5,71) и клещевина (5,30) среди бобовых получают сниженный фактор.

### Почему это важно для ккал

Фактор применяется к граммам белка, а те идут в ккал по 4. Ошибка белка переходит в ккал пропорционально:
орех с истинными 20 г белка при 6,25 залогируется как ~23,6 г → +14,6 ккал только с белка на 20 г; для
какао/грибов завышение ещё больше. Критично для белково-плотных позиций (протеиновые порошки, ореховые пасты,
соевые продукты), которые логируют по одной штуке — тут «в среднем по рациону разница вымывается» (FAO 2003)
НЕ работает.

### Рекомендация

- Хранить `NFactor` на курированный продукт/ингредиент (как поле `NFactor` в USDA FDC), дефолт 6,25.
- Применять факторы Джонса приоритетно для: **орехи/семена, соя, желатин, какао/шоколад, кофе, грибы, дрожжи,
  рафинированная пшеничная мука** (там 6,25 завышает ≥9%).
- Мясо/рыбу/яйца/кукурузу/большинство бобовых от 6,25 не отличать (Джонс им и даёт 6,25).
- «Крупная» наука (амино-кислотные факторы Mariotti k_A/k_P ≈5,6, новый общий дефолт) пока не принята ни USDA,
  ни INFOODS — это watch-item, не для хардкода.
- ⚠️ Точность: 6,25 как fallback оставить всегда, когда фактора нет. Истинный белок (сумма аминокислот) для нашей
  цепочки непрактичен (аминокислотных профилей в FatSecret/OFF нет) — crude protein (N×XN) это практический потолок,
  так же работает и сам USDA FDC.

## 4. Углеводы и жир — на каком базисе стандартизироваться

### Углеводы
Стандартизировать каноническое поле «углеводы» на **«всего по разности» (включает клетчатку)** — это то, что
реально лежит в >95% источников цепочки (USDA основное поле, этикетки OFF/FatSecret, российские таблицы). Ноль
конверсий для общего случая. Клетчатку **всегда хранить отдельным полем**, никогда не выводить вычитанием (кроме
известных UK/McCance-источников, где «углеводы» уже без клетчатки — низкочастотный кейс). «Усвояемые углеводы»
считать внутри (`всего − клетчатка`) только там, где нужно (гликемические дисплеи, наш собственный расчёт ккал).

### Жир
Стандартизировать на **total lipid / crude fat «как в источнике»** (USDA nutrient 204, OFF, FatSecret, курированное).
НЕ конвертировать между total-lipid и NLEA/триглицеридным базисом бланкетно:

- Для большинства логируемой еды (мясо/молочка/яйца/масла/орехи/злаки как едят) разрыв total-lipid vs NLEA = **0–13%** —
  меньше нашей же «≈»-неопределённости, не стоит поправки.
- Разрыв велик (15–40%+) для короткого списка: **субпродукты (печень), тощая/белая рыба, овощи-фрукты, злаки/мука** —
  там больше структурного/мембранного липида и невыжирных ко-экстрактивов. Если когда-то понадобится точность на этих
  категориях — точечная поправка по таблице Weihrauch/FAO A5.1, НЕ плоский «−4%» везде (для молочки/масел/орехов
  это будет неверно).

### Сахароспирты
Уже сидят внутри «всего углеводы» (и в EU, и в US конвенции). Но НЕ должны облагаться 4 ккал/г — им нужен свой
коэффициент. ТР ТС/EU: 2,4 ккал/г для всех, кроме эритрита (0). US FDA даёт по-своему на каждый (маннит 1,6;
изомальт/лактит 2,0; мальтит 2,1; ксилит 2,4; сорбит 2,6; HSH 3,0). Для RU-приложения держимся ЕАЭС: **2,4 общий,
эритрит 0**. Спец-кейс по имени: если в ингредиентах «эритрит/erythritol» → 0; иначе дженерик 2,4.
Продукт с 20 г «углеводов», из которых 15 г мальтита, — это ~20–36 ккал с этой части, а не 80. Большая ошибка,
если наивно считать сахароспирты обычными углеводами (важно для «кето»/«фитнес»-снеков).

## 5. Клетчатка (пищевые волокна) — определения, измерение, энергия, цель

### Определение
- **Codex Alimentarius (2009):** углеводные полимеры ≥10 мономерных единиц, не гидролизуемые эндогенными
  ферментами тонкого кишечника; решение включать ли 3–9 звеньев (олигосахариды/ФОС/инулин) — на усмотрение
  национальных органов. Отсюда два живых мировых стандарта (≥10 vs ≥3), и это одна из причин расхождений баз.
- **РФ терминология:** современный показатель — «пищевые волокна» (ферментативный метод). Старый советский
  показатель «клетчатка» (метод Веендера, «сырая клетчатка») ловит только целлюлозу/лигнин = **10–40% истинных
  пищевых волокон** → систематически ЗАНИЖЕН. Скурихин-2007 и база ФИЦ питания уже перешли на «пищевые волокна».

### Измерение — почему базы расходятся
| Метод | Что ловит | Проблема |
|---|---|---|
| Веендер («сырая клетчатка», сов.) | целлюлоза/лигнин | ловит 10–40% истины |
| Englyst / NSP (UK, McCance) | некрахмальные полисахариды | ИСКЛЮЧАЕТ лигнин и резистентный крахмал → систематически ниже |
| AOAC 985.29 / 991.43 (Prosky/Lee) | ВМ растворимая + нерастворимая | теряет резистентный крахмал, инулин, НМ-растворимую |
| AOAC 2009.01 / 2011.25 (McCleary) | + резистентный крахмал, ФОС/ГОС, полидекстроза | современный, физиологичный (37°C, pH 6) |

Одна и та же еда в разных базах может отличаться по клетчатке на **20–100%+** — не ошибка, а разные методы/традиции.
Больше всего расходятся: бобовые, цельные злаки, продукты с резистентным крахмалом (охлаждённый варёный рис/картофель/
паста, зелёный банан), инулин-содержащие (лук, чеснок, цикорий, артишок).

### Энергия клетчатки
**2 ккал/г (8 кДж/г)** — консенсус (FAO / EU 1169/2011 / FDA-2016 / ТР ТС). Основано на ~70% средней
ферментируемости в толстой кишке → короткоцепочечные жирные кислоты (Livesey 1990/1992; измеренный диапазон
1,5–2,5 ккал/г по типам волокна). «0» — устаревшая конвенция «сырой клетчатки». IOM осторожнее («точность
непредсказуема из-за микробиома»), но не оспаривает 2 как регуляторную конвенцию.

> Резистентный крахмал — обратная асимметрия: помечается как клетчатка и по US-этикетке = 0, но прямое измерение
> дало **~2,95 ккал/г**. Т.е. поправки на клетчатку режут в обе стороны, не всегда «цельная еда = меньше калорий».

### «Net carbs» (углеводы − клетчатка − сахароспирты)
НЕ выносить как основную метрику. Ни один регулятор (FDA/Codex/EFSA) термин не признаёт. Валидно вычесть клетчатку
(она не даёт глюкозу) и эритрит (~0), но сахароспирты не взаимозаменяемы (мальтит всасывается 40–60% и поднимает
глюкозу). Если когда-нибудь кето-фича — явно маркированное opt-in число `всего − клетчатка − 0,5·сахароспирты`
(клиническая «половинная» конвенция), не подменять «всего углеводы».

### Цель по клетчатке (суточная)
| Орган | Значение |
|---|---|
| IOM/NASEM (2005) | **14 г/1000 ккал** → М 19–50: 38 г; М 51+: 30 г; Ж 19–50: 25 г; Ж 51+: 21 г |
| EFSA (2010) | 25 г/сут (флэт) |
| WHO (актуально) | ≥25 г/сут |
| **РФ, МР 2.3.1.0253-21** | **20–25 г/сут, или 10 г/1000 ккал** (дети >1 г: 10–22 г) |

Самое резкое расхождение всего справочника: IOM масштабирует с калориями (при 3800 ккал → 53 г!), РФ/EFSA — флэт.
РФ-норма (20–25) заметно ниже международного консенсуса (25+).

**Рекомендация:** масштабировать от персонального бюджета, как уже делает приложение с другими целями:
```
fiberTargetG = max(25, round(calorieTarget / 1000 * 12))   // 12 ≈ середина РФ-10 и IOM-14
```
Для 1800–2800 ккал даёт ~22–34 г. Решение владельца: нужен ли пол ~18–20 г при агрессивном дефиците (источник
прямо не оговаривает). Пороги ТР ТС (Прил. 5) для бейджа «богато клетчаткой»: **источник** ≥3 г/100 г,
**высокое содержание** ≥6 г/100 г — официальная граница, а не произвольная.

### Пропущенная клетчатка — как обрабатывать
- НЕ забивать пустое поле нулём (классика → систематическое занижение; «импутированное значение почти всегда
  лучше нуля»).
- При fiber=0/пусто от менее надёжного источника (Branded/OFF/FatSecret) — тянуть клетчатку из следующего более
  надёжного звена для похожей еды, и НЕ давать «0,0 г» с этикетки перезаписать ненулевую оценку, полученную выше
  по цепочке. Целый «0» на злаке/бобовом — почти всегда округление (<0,5 г → «0»), а не истинное отсутствие.
- Наш LLM-fallback — это уже более умная стратегия закрытия пропусков, чем zero-fill; держать как последний слой.

## 6. Готовка — yield-факторы (сырой ↔ готовый)

### Одна формула, из которой всё следует
```
нутриент_на_100г_ГОТОВОГО = нутриент_на_100г_СЫРОГО × (Retention / Yield)
Yield = вес_после / вес_до
```
Для белка, углеводов и клетчатки **Retention ≈ 1,00** почти при любой обычной готовке (они не разрушаются) → для
макросов весь пересчёт сводится к делению/умножению на yield-фактор. **Исключение — жир** (см. ниже).

### Yield-факторы (хардкодить)

**Крупы/паста/бобовые (Bognár/FAO)** — то, что нужнее всего (кейс «сырой vs варёный»):

| Продукт | Yield (готовое/сухое) |
|---|---|
| Рис длиннозёрный шлифованный варёный | 2,98 |
| Рис бурый варёный | 2,60 |
| **Гречка варёная** | **3,60** |
| Перловка варёная | 2,50 |
| Овсянка/геркулес варёные | ~2,4 |
| **Лапша/макароны варёные** | **2,60** (яичная), 2,10 (без яйца, твёрдая пшеница) |
| Фасоль сухая → варёная | 2,50 |
| Чечевица сухая → варёная | 2,73 |

**Мясо/птица/рыба (USDA Cooking Yields, 2012)** — вес УБЫВАЕТ (уходит вода/жир):

| Продукт | Метод | Yield % |
|---|---|---|
| Говядина, тушение | braised | 67 |
| Говядина, гриль/жарка | broiled | 78 |
| Куриная грудка б/к | запекание | 72 |
| Куриная грудка | варка/припускание | 77 |
| Свиная вырезка | запекание | 80 |
| Фарш говяжий жирный (>22%) | broiled | 63 (жир −12 п.п.) |
| Бекон | жарка | 29–32 (жир −33…−36%) |
| Рыбное филе (треска/минтай) | варка/жарка | 77–80 |

**Овощи/картофель (Bognár):** картофель варёный очищенный 1,00; в мундире (съедобная часть) 0,80; печёный 0,68;
фри 0,49–0,60; морковь варёная 0,90–0,94; шпинат тушёный 0,76–0,84.

### Величина ошибки при путанице базиса
Ошибка = ровно yield-фактор или его обратная величина:

| Продукт | Yield | Сырой-базис на готовый вес | Готовый-базис на сырой вес |
|---|---|---|---|
| Рис варёный | 2,9 | **завышение до 290%** (почти ×3) | занижение до 34% |
| Паста варёная | 2,6 | завышение до 260% | занижение до 38% |
| Курогрудка запечённая | 0,75 | занижение до 75% | завышение до 133% |
| Говядина тушёная | 0,67 | занижение до 67% | завышение до 149% |

Это крупнейший по величине источник ошибки в пайплайне после массы порции — до ×3.

### Жир — единственный макрос с Retention ≠ 1,00
- **Жарка/фритюр (набор, не потеря):** панированное жареное мясо +5…+10 г/100 г; фри +4…+10 г/100 г; обычная
  жарка ~+1 г/100 г. Аддитивный член.
- **Жирный фарш/жирные куски сухим жаром со сливом:** реальный рендеринг — жирный (>22%) фарш теряет −12 п.п.
  жира, средний (12–22%) −5,2, тощий (<12%) −1,6. Для тощих кусков/птицы/рыбы/варки-тушения жир ≈ 1,00.

### Куда ложится в коде
- [`server/src/nutrition/dryBasis.ts`](../server/src/nutrition/dryBasis.ts) уже ловит именно этот класс ошибок
  (сухой крахмал × вес готового = ~3× завышение, «1400 ккал на 350 г»). Философия «показываем факт, не переписываем
  числа втихую» — верная (raw→cooked слишком специфичен для еды). Можно РАСШИРИТЬ: вместо только флага давать
  yield-подсказку / предлагать «варёную» строку с делением на фактор.
- **Правило:** всегда матчить базис нутриента (сырой vs готовый) к базису логированного веса. Фото/зрение — ВСЕГДА
  оценка готового/поданного состояния; если матч попал в сырую строку — применить yield перед ккал, иначе ×2–3.
- **LLM-путь:** просить модель рассуждать и отдавать числа сразу в состоянии «как съедено» (готовое) — это обходит
  всю yield-арифметику для fallback-оценки.
- Тэг состояния (raw/dry/cooked-by-method) на каждую запись БД — корень проблемы в том, что «рис» лежит без тэга
  состояния, и матчинг молча ранжирует к ближайшей строке. Дефолт по категории: крупы/паста/бобовые → всегда
  cooked; мясо/рыба в дневнике → cooked (люди логируют тарелку), сырой только при явном «сырой/сырая» или
  рецептурном контексте.

### Микронутриенты (на будущее)
Если появятся витамины/минералы — НЕ переносить макро-допущение Retention≈1. По USDA Retention Factors R6
вар-и-слить сильно бьёт по калию (паста слитая — 30% удержания) и витамину C (варёная зелень — 55%).

## 7. Метаболизируемая энергия — пределы Атвотера

Факторы Атвотера откалиброваны под **смешанный западный рацион** и в агрегате для него точны (потому и живут
120+ лет). Но контролируемые кормовые RCT (USDA-ARS, бомбовая калориметрия + полный сбор кала/мочи) показывают
крупные, воспроизводимые, продукт-специфичные завышения:

### Цельные орехи и бобовые (форма — главная переменная)

| Продукт / форма | Измеренная ME | Атвотер | Завышение |
|---|---|---|---|
| Миндаль цельный сырой | 4,4–4,6 ккал/г | 5,9–6,1 | **24–32%** |
| Миндаль цельный жареный | 4,86 | 6,01 | 19% |
| Миндаль рубленый жареный | 5,04 | 6,04 | 17% |
| **Миндальная паста (молотый)** | 6,53 | 6,18 | **~0** |
| Грецкий орех | 5,21 | 6,61 | 21–22% |
| Фисташки | 5,40 | 5,66 | ~5% |
| Кешью | 4,89 | 5,82 | 14–16% |
| Нут (варёный, цельный) | 123 ккал/порц | 134–137 | 8–10% |
| Чечевица (варёная) | 119 | 138–142 | 14–16% |

**Механизм:** клеточные стенки растений не рвутся при жевании — внутриклеточный жир заперт и недоступен липазе.
Поэтому поправка **монотонно исчезает с разрушением формы**: цельный сырой > цельный жареный > рубленый >
молотый/паста (≈0). ⚠️ Точные % миндаля (Novotny 2012) — full-text не открылся у агента; качественный факт и
цитата надёжны, точную цифру сверить перед хардкодом.

### Рекомендация
- Опционально: keyed по ФОРМЕ поправка для цельных орехов/семян (~0,68–0,76× для цельных сырых, ~0,80–0,85×
  жареных, ~0,83–0,88× рубленых, **без скидки** для паст/муки/молока). Требует, чтобы матчинг различал форму
  («миндаль целый» vs «миндальная паста») — реально при нашем парсинге.
- Осторожнее то же для цельных бобовых (нут/чечевица) −8…−16% vs без скидки для хумуса/муки/изолята.
- **TEF (термический эффект пищи)** — реален и зависит от макроса (белок 20–30%, углеводы 5–10%, жир 0–3%), но это
  свойство рациона/человека, а не продукта. НЕ моделировать per-food. Наш адаптивный TDEE по тренду веса уже
  эмпирически впитывает устойчивую персональную TEF/absorption-погрешность — это более надёжное место, чем
  нестабильный множитель в записях еды.
- **Инженерная проверка (самый дешёвый и надёжный фикс):** убедиться, что клетчатка нигде не пересчитывается по
  4 ккал/г (см. §1). Если ккал где-то считается как `белок·4 + жир·9 + всего_углеводы·4` вместо доверия к
  fiber-aware числу — волокнистое систематически завышается.
- Все первичные RCT — малые выборки (n=16–18, в основном одна лаборатория), большой межиндивидуальный разброс.
  Это популяционные поправки, не гарантия для конкретного пользователя → раскрывать в «Как это работает», не
  обещать ложную точность (совпадает со стилем «≈»).

## 8. Порции и плотность — крупнейший источник ошибки

`ккал = масса_г × ккал/г` → ошибка массы переходит в ккал **1:1**. Это множитель поверх (более точных) табличных
чисел на грамм, поэтому оценка массы — самый высокий рычаг точности.

### Дефолты порций (RACC / FNDDS, хардкодить как приоры)

| Категория | RACC-дефолт | Мера |
|---|---|---|
| Хлеб ломтик | 50 г | 1 ломтик |
| Крупа/паста готовая | 140 г | 1 чашка |
| Яйцо | 50 г | 1 крупное |
| Мясо/рыба порция | 85 г готового (≈«колода карт»/ладонь) | 3 oz |
| Овощи без соуса | 85 г | ½–1 чашка |
| Фрукт | 140 г | 1 шт/чашка |
| Бобовые готовые | 90–130 г | ½ чашки |
| Суп | 245 г | 1 чашка |
| Соусы (спагетти) | 125 г | ½ чашки |
| Соусы/заправки минорные, масло/сливочное | 15 мл (1 ст.л.) | **не оценивать визуально** |
| Орехи/семена | 30 г | горсть |
| Напитки/сок | 240 мл | 1 чашка |

⚠️ RACC — регуляторные нормы потребления, не измеренные веса чашки. Реальная чашка варёного риса ~155–160 г
против RACC 140 г (~10–15% разрыв). Использовать как «дефолт-когда-ничего-не-известно», предпочитать
FDC/FNDDS-конкретику при матче.

### Плотность (объём → масса; >20× разброс на ту же чашку!)
| Продукт в «чашке» (240 мл) | Плотность г/мл | ~Масса |
|---|---|---|
| Листовой салат/шпинат сырой | 0,06–0,08 | ~14–19 г |
| Сухие хлопья | 0,10–0,18 | ~24–43 г |
| Варёный рис | 0,70–0,73 | ~170 г |
| Варёная паста | 0,55–0,59 | ~132–142 г |
| Молоко/вода | ~1,00 | ~240 г |
| Мёд | 1,42 | ~341 г |
| Сахар | 0,88–0,90 | ~211 г |

**Никогда не конвертировать «чашку»/визуальный объём в граммы без плотности продукта (хотя бы категории).**
Для неопознанного варёного смешанного блюда защитимый единый дефолт — **0,85 г/мл**. Источник полных таблиц:
**FAO/INFOODS Density Database v2.0** (638 позиций, 20 групп).

### Ошибки оценки (количественно)
- **Потолок ~50–65%** оценок в пределах ±25–30% истины даже у обученных ассессоров с фото-референсами.
- **Contraction bias** (устойчив во всех исследованиях): малые порции переоцениваются (+20…+90%), большие
  недооцениваются (−16…−22%), средние — точно. Если оценщик (человек или ИИ) говорит «большая порция» — истина
  вероятно ещё больше, и наоборот.
- **Соусы/масло/заправки — худшая категория** (относительные ошибки 100–250%): малые абсолютные количества →
  любая фикс-ошибка = огромный %. Не давать зрению задавать их — дефолт-референс (1 ст.л.) или спросить.
- **Аморфное** (пюре, рис, каши, супы) — систематически хуже дискретно-счётного (ломтики, яйца, стейки). Для
  последнего счёт штук + вес/штуку точнее оценки объёма.
- **Текст с количеством часто бьёт фото** (Lucassen: 50% vs 35% в ±25%). Поддерживает наш гибрид: если пользователь
  назвал вес/количество — верить ему, а не переоценивать по фото.
- Возраст (65+) и высокий ИМТ ухудшают точность на ~10 п.п. — быть консервативнее (шире «≈») для этих сегментов
  (совпадает с нашим нуджем при ИМТ≥30).

### Полосы неопределённости (для ширины «≈»)
| Категория | Дефолт | Неопределённость |
|---|---|---|
| Дискретно-счётное (яйцо, ломтик, фрукт, банка) | RACC/FNDDS вес штуки | ±15–20% |
| Крупа/паста/картофель, «чашка» | RACC 140 г × плотность | ±25–30% |
| Смешанное блюдо неопознанное | 0,85 г/мл × объём | ±30–40% |
| Мясо/рыба (опознанный кусок) | RACC 85 г | ±20–25% |
| Овощи россыпью | RACC 85 г | ±25–35% |
| Соусы/заправки/масло | RACC 15/5 мл, не оценивать визуально | ±50–100% визуально; ±20% если подтверждён счёт |
| Напитки поштучно | RACC 240 мл / по таре | ±30–50% фото |
| Суп | RACC 245 г, плотность ~1,0 | ±20–25% (близко к воде → надёжно) |

### Лучшая архитектура оценки
«Референс-объект (тарелка/рука/утварь) + формула формы + плотность из таблицы» > «прикинуть объём в чашках»
(80% в ±25% против 29% для геометрически регулярной еды). «Лестница порций» — геометрической прогрессией
(×1,5 шаг: 22, 33, 50, 74, 112, 168, 251 г), а не фикс-граммами.

- IOM 25% добавленного сахара — это устаревший «толерантный максимум», НЕ цель; WHO и РФ согласны на <10%.
  Не выносить IOM-число как конкурента — оно выбьет консенсусную цифру в аутлаеры.
- Клетчатка — самое резкое расхождение (см. §5): IOM масштабирует с калориями, РФ/EFSA — флэт.
- Белок 0,8 vs 0,83 г/кг — разница конвенции округления (обе от EAR 0,66), не науки; ~2 г/сут на 70 кг. Можно
  стандартизироваться на 0,83 (к нему сошлись РФ, EFSA, WHO/FAO/UNU-2007).
- Пожилые (PROT-AGE): 1,0–1,2 г/кг; активные пожилые ≥1,2; при болезни 1,2–1,5. Спорт (ISSN): 1,4–2,0 г/кг,
  на сушке до 2,3–3,1.

### PAL ↔ КФА кросс-уок (переиспользовать один концепт для двух пикеров)
FAO/WHO/UNU 2004: сидячий 1,40–1,69; активный 1,70–1,99; интенсивный 2,00–2,40 (>2,40 не удержать долго; ≥1,75
«желательно» для снижения NCD). Соответствие: КФА 1,4 (умственный труд) → низ сидячего; КФА 1,6 (лёгкий труд) →
верх сидячего; КФА 1,9 (средний труд) → верх активного; КФА 2,2 (тяжёлый труд) → интенсивный. 65+ (РФ 1,7) — на
границе сидячий/активный.

⚠️ **Ловушка:** IOM-коэффициенты `PA` (сидячий=1,00) и FAO PAL-множитель (сидячий 1,4) НЕ взаимозаменяемы, хотя
названия похожи. IOM PA=1,00 уже включает бытовую активность в константы регрессии; PAL=1,0 = постельный режим.
Не смешивать константы двух систем. ⚠️ Точные IOM PA-константы у агента взяты из вторички, не сверены с NAP.edu —
подтвердить перед хардкодом.

### Формула % → граммы
`граммы = (ккал_цель × доля_макроса) / плотность` (4 ккал/г белок/углеводы, 9 жир). Проверка: РФ М КФА1.4 18–29:
2400 × 30% / 9 = 80 г жира ✓ (совпадает с таблицей МР).

## 10. ИИ-оценка по фото/тексту — честность «≈»

### Реалистичный потолок
Даже фронтир-модели (GPT-4o/4.1, Claude 3.5/3.7) дают **~24–38% MAPE по ккал с одного фото** — вровень с
диетологами (~25–44%), лучше любителей (~50–58%), но далеко не «измерение». Бенчмарки:
- **Nutrition5k (Google, CVPR 2021):** только ID+плотность = 9,5% MAE ккал; + оценка массы = 26,1% (RGB) /
  16,5% (RGB-D). Оценка массы одна стоит 7–17 п.п. — крупнейшая компонента.
- **ChatGPT-5, 195 блюд:** фото 30,5% MAPE → фото+метаданные 24,4% → фото+детальный текст с весом **13,9%** →
  только текст с весом 18,1%. **Текст с весом бьёт фото-only.**
- Все модели: недооценка РАСТЁТ с размером порции (bias slope −0,23…−0,50) — та же большая-порция-предвзятость,
  что у людей.

### Где доминирует ошибка (тот же вывод, что §2/§8)
**масса/порция (крупнейшая, 15–30+ п.п.) > идентификация составных/незнакомых блюд (борщ, солянка, оливье, плов —
мультикомпонент, окклюзия, жидкость) > поиск плотности при верном ID (наименьшая, ~9–15%, где хорошая курированная
база отбивает своё против угадывания LLM).**

### Расширять «≈» когда:
фото без веса; блюдо составное/жидкое; порция крупная; жареное/жирное (невидимый жир — худший макрос у всех
моделей); нет массы вообще.
### Сужать «≈» когда:
пользователь ввёл вес («200 г творога 5%») — схлопывает главный источник ошибки; уверенный курированный/этикеточный
матч (наше «Как в базе: {name}·{source}»); еда одиночная/однородная/счётная (яйцо, банан, ломтик).

### Подача (честность неопределённости)
- Точная точность читается как ложная уверенность (Jerez-Fernandez 2014): «417 ккал» на фото-догадке с реальным
  ±25–35% вводит в заблуждение. Округлять фото-ккал до ближайших **10–25 ккал**, не до 1.
- Числовой диапазон почти не роняет доверие (van der Bles 2020, PNAS); расплывчатые словесные хеджи — хуже.
  Для низкой уверенности предпочесть диапазон/округлённую «≈середину» одиночному числу.
- Держать сам квалификатор коротким и единым по приложению («≈ проверьте»), а не числовым виджетом уверенности.
- Ширину «≈» варьировать по уровню улики (фото-only vs фото+текст vs текст+вес vs верифицированный матч) — ошибка
  реально меняется 2×+, единый «±20%» везде — сам по себе ложная точность в другую сторону.
- Периодически перепроверять точность на актуальной live-модели: единственное академ-сравнение нашло семейство
  Gemini заметно позади GPT-4o/Claude (на старом чекпоинте) — повод к спот-чекам, не вердикт текущей модели.

Совпадает с уже реализованным: `matched_name`/«Как в базе», «Вес не указан — прикинул N г», «≈ проверьте».

- TEF/NME per-food — не моделировать (адаптивный TDEE уже впитывает).
- Не переписывать raw→cooked числа втихую — только флаг/подсказка (как сейчас в `dryBasis.ts`).
- Истинный белок (сумма аминокислот) — непрактичен для нашей цепочки, crude protein — потолок.
